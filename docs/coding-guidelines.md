# コーディング方針

## エラーハンドリング

基本的にtry-catch,begin-rescueなどエラーハンドリング構文の記述は禁止です。構文を書くときは必ず次の要件を満たすときにのみ許可します。

- 当方では解決困難なエラーが発生する具体的な候補がある。外部APIエラーなど

そして、条件を満たしており構文を書くべきときは必ず次のルールに従うこと。

- 具体的な解決困難なエラーの想定内容をリストアップしてコメントとして記述し、何故エラーハンドリングが必要か明文化する
- 元々のオリジナルのエラーの全情報を表示することで、エラーハンドリングのせいでエラー情報が欠損させない
- エラーの原因となった変数などの情報も可能な限り標準エラー出力する
